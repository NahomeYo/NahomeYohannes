Problem Description

The longest common subsequence algorithm finds the length of the longest sequence of characters that appears in both input strings while preserving order. The characters do not need to be adjacent, which makes a subsequence more flexible than a substring. For that reason, two strings can share an LCS even when the matching characters are separated by many other characters in between.

This problem is a classic dynamic programming example because many smaller prefix comparisons repeat across the full problem. Instead of solving those same comparisons again and again, the algorithm stores partial answers in a table and builds the final result from them.

Algorithm Steps

  • Create a dynamic programming table with m + 1 rows and n + 1 columns.

  • Use the first row and first column as base cases for comparisons against an empty prefix.

  • Fill the table row by row.

  • If the current characters match, set the current cell to the diagonal value plus 1.

  • If the current characters differ, set the current cell to the larger of the value above and the value to the left.

  • Return the bottom-right value as the LCS length.

Explanation

Each entry dp[i][j] stores the LCS length for the first i characters of the first string and the first j characters of the second string. That means every cell represents a smaller prefix version of the original problem, which is why a table-based dynamic programming approach works so well here.

When the current characters match, the subsequence can grow by one from the diagonal subproblem, so the algorithm uses dp[i - 1][j - 1] + 1. When the characters do not match, the algorithm cannot extend a common subsequence at that position, so it takes the better of two smaller possibilities: skip the current character from the first string or skip the current character from the second string.

In JavaScript, the 2D table is built with Array.from() and fill(0), which makes it easy to create the row-and-column structure for the dynamic programming process. When characters do not match, the code uses Math.max() to pick the better of the two smaller subproblems, so the recurrence is written very directly in the implementation.

Because the table has (m + 1)(n + 1) cells and each cell takes constant time to compute, both the runtime and the space complexity are O(mn). This is one of the clearest examples of how dynamic programming converts repeated recursive comparisons into an organized table-filling process.
